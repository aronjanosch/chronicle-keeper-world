    Settings

    Backend URL and data folder are required once. Everything else pre-fills the Transcribe and Summarize dialogs; change a field there only when you want a one-off override.

    App & storage

    Where the Chronicle Keeper API is running. Stored in this browser only.

    Root directory for campaigns, sessions, transcripts, and exports. Use an absolute path the backend can read and write.

    Transcription defaults

    Transcription runs on-device via the built-in Parakeet engine (sherpa-onnx). Auto and Parakeet are equivalent today; the explicit option is there for future engines.

    Opt-in execution provider for on-device transcription. CPU is already several× realtime; the others only help if your build's onnxruntime supports them — otherwise the engine silently falls back to CPU.

    Pre-fills Transcribe and is the language passed to the summarizer (BCP-47, e.g. en, de).

    Used when you transcribe without choosing a model in the session dialog. Defaults to nemo-parakeet-tdt-0.6b-v3. If it does not match a known model, the engine falls back to its built-in default (Parakeet v3).
